# docs

Long-form workspace documentation that doesn't fit the per-directory AGENTS.md format
or the README.

## Files at this level

- `JOURNEY.md` — the narrative history of the project: monolith → branded six-repo
  split → dormancy → audit → monorepo consolidation → live verification incidents →
  publication. Every incident it describes is real and traceable to commits, tests, or
  code comments. When a significant new chapter happens (major refactor, notable
  incident, new capability), append to it rather than rewriting history.

Human-facing prose lives here; agent-facing operational docs stay in the AGENTS.md
tree; system-wide rules stay in `.kiro/steering/`.
